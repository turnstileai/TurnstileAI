# Contributing

Thanks for your interest in contributing to TurnstileAI.

## Ways to contribute

- Report bugs.
- Suggest improvements.
- Improve documentation.
- Add examples.
- Submit code changes.
- Help expand tests.

## Before you start

Please check existing issues and pull requests before opening a new one. This helps avoid duplicate work.

## Development setup

1. Clone the repository.
2. Install dependencies.
3. Build the project.

```bash
npm install
npm run build
```

## Project structure

- `src/` contains the SDK source code.
- `docs/` contains project documentation.
- `examples/` contains usage examples.
- `tests/` contains test coverage.

## Branching

- Create a new branch for each change.
- Keep pull requests focused on one thing.
- Use clear branch names such as `fix/auth-error` or `docs/quickstart-update`.

## Pull requests

Before opening a pull request:

- Make sure the code builds.
- Add or update tests where relevant.
- Update docs if behaviour changes.
- Keep commit messages clear and specific.

## Issues

When opening an issue, include:

- A short summary.
- Steps to reproduce.
- Expected behaviour.
- Actual behaviour.
- Environment details, if relevant.

## Code style

- Keep changes small and readable.
- Prefer clear names over clever code.
- Avoid unrelated refactors in the same pull request.

## Documentation changes

Documentation improvements are welcome. If you update an API, example, or workflow, please update the relevant docs too.

## Questions

If something is unclear, open an issue and explain what you are trying to do.
